linux定时任务的设置 crontab

前端时间因为项目的需求，要写一些定时执行的任务，就用到了Linux的crontab。其原理就是定时执行我们规定要的文件。
几个基本的说明：

1.基本命令：

crontab -e : 执行文字编辑器来设定时程表（编辑crontab）。
crontab -r : 删除目前的时程表
crontab -l : 列出目前的时程表
crontab file -u user-用指定的文件替代目前的crontab。

2. crontab 的格式说明：基本格式 ：

※　　※　　※　　※ 　 ※ 　command
分　 时　 日　 月　 周　 命令
第1列表示分钟1～59 每分钟用～～ ～表示
第2列表示小时1～23（0表示0点）
第3列表示日期1～31
第4列表示月份1～12
第5列标识号星期0～6（0表示星期天）
第6列要运行的命令
crontab文件的一些例子：
30 21 ※ ※ ※ /usr/local/etc/rc.d/lighttpd restart

上面的例子表示每晚的21:30重启apache。

3.项目例子:

php /data/web/service.snippetnews.com.ph/dev/D0.0.10/cli/index.php PushMsg index >> /data/logs/PushMsg.log 2>&1(日志位置)

mac install yaf

之前在win使用过yaf框架，第一次在mac安装yaf扩展，所以要记录下。

1. 下载 yaf 最新的源代码wget http://pecl.php.net/get/yaf-3.0.2.tgz

2. 解压（注明：要在下载文件的目录下）tar zxvf yaf-3.0.2.tgz && cd yaf-3.0.2

3. 安装yaf扩展（注明：要在解压后的文件目录内下）

1: /usr/local/bin/phpize（注明：/usr/local/bin/ 是PHP的bin目录）
2: ./configure –with-php-config=/usr/local/bin/php-config
3：make
4：make test
5：sudo make install

4. php.ini添加yaf.sovim xxx/php.ini (注明：/usr/local/etc/php/7.0 是PHP的php.ini 目录)

在php.ini文件最后添加 yaf.so
extension=yaf.so

5.重启PHP

killall php-fpm
php-fpm -D

git开发使用及部署

记录在开发过程使用git的一些基本命令。

1.git 开发使用步骤git clone git url

git clone ssh://XXX.bbtu.com@10.8.8.61:22/xxx.bbtu.com.git
开发分支: develop
进入到git目录
cd /git/xxxxxx

步骤：

1.切换到develop分支

git checkout develop

2.pull develop 分支

git pull origin develop

3.创建新分支并切换到新分支

git branch new branch
branch name 使用本次需要开发的功能命名，例如order
git branch order-branch
git checkout order-branch

4.在新分支中开发

5.开发完成后跑测试用例

6.commit 新分支git commit -m “开发项目描述”

7.切换回develop分支

git checkout develop

8.pull develop 分支(获取其他开发提交的代码)

git pull origin develop

9.合并新分支到develop分支，修改冲突部分

git merge new branch
git merge order-branch

10.合并后跑测试用例（测试合并后代码）

11.commit and push

git add .
git commit -m “merge xxxx”
git push origin develop

2.部署完成一个要上线的版本，部署

1.打taggit tag tag name

git tag R1.3.50

2.推taggit

push origin tag name
git push origin R1.3.50

3.重推tag删除已存在tag

git tag -d tag name
git push origin –delete tag tag name
git tag -d R1.3.50
git push origin –delete tag R1.3.50

4.重新打tag推tag

git tag R1.3.50
git push origin R1.3.50

5.查看已存在tag

git tag

3.已上release的项目需要

hot fix

1.获取对应版本的tag 代码

git checkout tag name
git checkout R1.3.50
创建hot fix分支,切换到
hot fix 分支
git branch hot-fix-branch
git checkout hot-fix-branch

2.修改代码

3.提交代码

git commit -m “修改描述”
在hot fix branch打tag,
推tag部署到release
把hot fix branch合拼到develop

建博客心得

很开心经过几天的折腾，总于把自己的技术博客搭建起来了。用了Hexo博客框架。
搭建的过程中遇到了不少问题。有些目前还没解决，先让博客运行起来。以后再慢慢解决各个问题吧。

主要笔记：

1._config.yml配置-多平台配置deploy:

type: git
repo:
github: https://github.com/dengxiaowu/dengxiaowu.github.io.git,master
gitcafe: https://git.coding.net/dengxiaowu/dengxiaowu.git,master

2.三个基本的部署命令

hexo clean
hexo generate
hexo deploy

3.域名解析

我是在万网购买的域名，万网里的NDS解析是收费的，所以可以选择DSNlop或者Cloudxns 免费域名解析平台！注意域名要实名认证！
注意：
我用的是Cloudxns，比较稳定，还可以监控，总体来说非常不错！github.io 的部署解析后记得要在source/需要创建一个CNAME文件，填入自己个人的域名(不需要加上http)！但是我部署了coding，域名老是解析失败，后来停掉了。以后要解决这个问题。
